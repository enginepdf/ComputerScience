인터넷 : 수십억 대의 컴퓨터를 모두 연결하는 기술 인프라, 서버와 클라이언트가 TCP/IP를 이용해 정보를 주고 받음

경로 : 컴퓨터 - 라우터 - 모뎀 - 전화시설 - ISP - 전화시설 - 모뎀 - 라우터 - 컴퓨터

네트워크 : 컴퓨터들의 연결(케이블 이용한 물리적 연결, 무선)

라우터 : 주어진 컴퓨터에서 보낸 메시지가 올바른 대상 컴퓨터에 도착하는지 확인(단 하나의 작업만 함)
       라우터와 라우터도 서로 연결 가능(무한히 확장 가능)
       일종의 컴퓨터

모뎀(MODEM, MOdulator and DEModulator) : 네트워크의 정보와 전화 시설에서 처리할 수 있는 정보를 상호교환하게 해줌(광섬유 케이블의 사용으로 전송 속도 향상)

ISP(Internet Service Provider) : 인터넷 서비스 제공 회사, 특수한 라우터 관리하고 다른 ISp의 라우터에도 액세스 할 수 있는 회사

IP / 도메인 : 컴퓨터 구분 위한 고유 주소, 서버와 클라이언트가 데이터를 주고 받는 규약인 IP(도메인을 이용해서 인간 친화적 사용)

웹 : 인터넷 인프라 위에 구축된 서비스(이메일 등도 있음)

MAC 주소(Media Access Control Address) : 네트워크 세그먼트의 데이터 링크 계층에서 통신을 위한 네트워크 인터페이스에 할당된 고유 식별자

유동IP가 MAC주소를 이용해 ip 할당, 컴퓨터나 공유기 등의 장치마다 고유주소(MAC 주소 - 물리적 주소)를 가지고 있어서 연결 장치마다 다른 IP 할당

NAT(Network Address Translation) : 공유기가 하나의 공인 IP를 여러개의 내부망 사설 IP들로 나누는 기술, 내부망을 공유기 외부로부터 보호.

게이트웨이(Gateway)

       컴퓨터 네트워크에서 서로 다른 통신망, 프로토콜을 사용하는 네트워크 간의 통신을 가능하게 하는 컴퓨터, 소프트웨어
       다른 네트워크로 들어가는 입구 역할. 종류가 다른 네트워크 간의 통로 역할(서로 다른 네트워크 상의 통신 프로토콜을 적절히 변환해주는 역할)
       게이트웨이를 지날 때마다 트래픽 증가하여 속도가 느려질 수 있음
       하나 이상의 프로토콜을 사용하여 통신한다는 면에서 라우터, 스위치와 구별. OSI 참조 모델의 전 계층에서 동작 가능(전송 방식이 다른 통신망을 흡수하여 서로 다른 기종끼리도 접속 가능하게 함)
       OSI 1~3 계층 : 라우터가 역할 맡음, OSI 4~7 계층에서 상이한 프로토콜들 간의 특수한 변환 담당하는 복잡한 소프트웨어를 수행하는 서버를 의미하기도 함
       집 -> 공유기 -> 인터넷 제공 회사 라우터 -> 인터넷망(공유기와 인터넷 제공 회사의 라우터는 이전 단계에서 다음 단계로 넘어 갈 때의 게이트웨이 역할 담당. 게이트웨이의 수를 Hop Count라고 함. 이 경우 인터넷 연결할 때까지 필요한 Hop Count===2)
