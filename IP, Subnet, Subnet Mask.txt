* IP(Internet Protocol) : 기기 간 네트워크 통신하는 방식

IP에서 IP 기기의 주소를 나타내는 것이 IP 주소(데이터를 정확히 송수신하기 위함)

IPv4 : 32bit로 처리, 옥택당 2^8=256개의 수 나타냄, 총 2^32만큼 IP 할당 가능  ex) 192.168.0.1
IPv6 : 128bit로 처리, 총 2^128만큼 IP 할당 가능, ex) 2001:0db8:0dad:0010:0000:0000:0000:0100


  * IPv4 클래스 분류
            첫번재 옥텟 | 네트워크 비트 | 호스트 비트 | 네트워크 수 | 호스트 수 | 기본 서브넷 마스크
    A Class (0)000 0000 | 8 | 24 | 128(2^8) | 16777214 | 255.0,0,0, 0000 0000 ~ 0111 1111(0 ~ 127) 0.0.0.0 ~ 127.255.255.255 - 대규모 네트워크 환경(약 1600만개)
    B Class (10)00 0000 | 16 | 16 | 16384 | 65534 | 255.255.0.0, 1000 0000 ~ 1011 1111(128 ~ 191) 128.0.0.0 ~ 191.255.255.255 - 중규모 네트워크 환경용(약 6만 5천개)
    C Class (110)0 0000 | 24 | 8 | 2097152 | 254 | 255.255.255.0, 1100 0000 ~ 1101 1111(192 ~ 223) 192.0.0.0 ~ 223.255.255.255 - 소규모 네트워크 환경용(254개)
    D Class (1110) 0000 | N/A | N/A | N/A | N/A, 1110 0000 ~ 1110 1111(224 ~ 239) 224.0.0.0 ~ 239.255.255.255 - 멀티캐스트
    E Class (1111) 0000 | N/A | N/A | N/A | N/A, 1111 0000 ~ 1111 1111(240 ~ 255) 240.0.0.0 ~ 255.255.255.255 - 연구/개발용, 기타

  * Network ID : 모든 Host 관리가 어렵기 때문에 Network 범위 지정해서 호스트 집합 관리, 인터넷 상에서 네트워크 구분하는 역할
    Host ID : 호스트 개별적 관리, 특정 네으퉈크 안에 존재하는 기기들을 구분하는 역할    --> Network ID + Host ID === IP 주소
    브로드캐스트 도메인 : 라우터, 네트워크 장비 없이 통신 가능한 영역
    예약된 IP 주소 : 127.0.01(loop back), 192.168.0.0(사설 네트워크),  224.0.0.0(멀티 캐스트), 240.0.0.0(미래 사용 용도 위한 예약)
    인터넷 공유기는 가상 IP 주소를 생성(NAT 서비스, 192.168.xxx.xxx)하여 각 컴퓨터 등에게 자동 할당(DHCP)하는 역할 수행
    각 Network ID에서 첫번째 주소는 Network Address, 마지막 주소는 Broadcast로 쓰여 총 사용 가능한 IP 주소 갯수 - 2
    같은 네트워크에 있다는 것은 라우터를  통하지 않고도 통신할 수 있는 같은 Network ID를 가지는 망에 있다는 것을 뜻함

* Subnetting

네트워크 관리자가 네트워크 성능 향상을 위해 자원을 효율적으로 분배하는 것(네트워크 영역과 호스트 영역 분할)
IPv4 기준 2^32만큼 표현 가능, 브로드캐스트 도메인에서 IP끼리 통신 위해 네트워크 영역 같아야하고 호스트 IP  달라야 함
하나의 주소에 서브넷 마스크를 씌워 자신에게 맞는 네트워크 만드는 것
A Class 네트워크의 경우 큰 네트워크를 작게 나누어 쓰지 않는 IP주소 자원을 최소화하려는 것이 주된 목적
브로드 캐스팅(특정 네트워크에 연결된 모든 기기에 하나의 메시지로 동시에 데이터 보냄)을 할 때 연결된 호스트 디바이스가 많으면 성능의 저하가 올 수 있어 네트워크의 규모를 줄여 크로드 캐스팅으로 인한 부하를 줄임


* Subnet Mask

서브넷 마스크 값은 네트워크 주소를 나타내는 비트 수를 명시한 값
1, 0으로 구성. 왼쪽(최상위 비트)부터 1이 연속으로 있어야 함  

ex) 192.168.0.1/24  C 클래스, 디폴트 마스크 255.255.255.0 -> 1111 1111.1111 1111.1111 1111.0000 0000    
    ->1(네트워크 영역)이 24개, 0(호스트 IP)

서브넷 마스크가 255.255.255.0일 때 네트워크가 가질 수 있는 호스트 주소의 수는 256(2^8)
-->  이를 두 개의 네트워크로 분할하고 싶을 때 서브넷으로 분할 xxxx xxxx.xxxx xxxx.xxxx xxxx.1xxx xxxx + xxxx xxxx.xxxx xxxx.xxxx xxxx.0xxx xxxx  --> 서브넷 마스크 /25, 255.255.255.128
     200.0.1.129라는 IP 주소는 200.0.1.129 ~ 200.0.1.255 라는 네트워크에 속한 호스트의 IP주소(200.0.1.129/25)

--> 150.100.100.1(B 클래스)에 서브넷 마스크 255.255.255.0(C 클래스 기본 서브넷 마스크) 쓸 경우, B 클래스 주소를 C 클래스 주소처럼 사용하겠다는 뜻으로, 150.100.100까지 네트워크 영역이 되고 마지막 옥텟만 호스트 IP 배정
    네트워크  영역을 늘리고 호스트 영역을  줄임
    (서브넷 마스크로 나뉘어진 서브넷 네트워크(독립적 네트워크) 간 통신은 라우터를 통해 통신)

--> 한 Bit씩 서브넷  마스크에 추가할 때마다 네트워크의 크기는 2배 증가하고 할당할 수 있는 호스트의 수는 2로 나뉘어 짐

* Subnet Network

기본 서브넷 마스크로 쪼개진 네트워크 주소를 서브넷 네트워크라고 함
IP 주소와 서브넷 마스크를 논리 AND 연산하면 구할 수 있다.
C 클래스의 경우 192.0.0 ~ 223.255.255까지의 네트워크가 곧 서브넷 네트워크
