- 프로세스 : 프로그램의 인스턴스. 실행 중인 프로그램으로 디스크로부터 메모리에 적재되어 CPU의 할당을 받을 수 있는 것. 운영체제로부터 주소 공간, 파일, 메모리 등을 할당.
           프로세스 스택(함수의 매개변수, 복귀 주소와 로컬 변수와 같은 임시 자료), 데이터 섹션(전역 변수), 힙(프로세스 실행 중에 동적으로 할당되는 메로리) 등을 포함
    PCB(Process Control Block) : 프로세스에 대한 중요 정보를 저장하고 있는 운영체제의 자료구조. 프로세스 생성과 동시에 고유 PCB 생성. PID(Process ID), process state, program counter, cpu register, cpu 스케쥴링 정보, 메모리 관리 정보, 입출력 상태 정보, 어카운팅 정보 등으로 구성

- 쓰레드 : 한 프로세스의 다양한 실행 흐름, 프롷세스의 실행 단위. 프로세스 생성 시 Main 쓰레드 하나 생성. 각 쓰레드는 각각의 스택 공간을 제외한 나머지 부분을 공유.
         각 쓰레드는 독립적인 작업을 수행해야 하기 때문에 각자의 스택과 PC 레지스터 값(쓰레드가 명령어의 어디까지 수행했는지 나타냄)을 갖음.

- 멀티 쓰레드 장점 : 멀티 프로세스에 비해 메모리 및 시스템 자원 소모 매우 낮음. 쓰레드간 통신 용이 ( 전역 변수로 쉽게 통신 )

- 멀티 쓰레드 단점 : 전역변수를 이용한 통신은 충돌 문제를 지니고 있다(서로 다른 쓰레드가 데이터와 힙 영역을 공유). 동기화 문제를 해결해야 함.

- 멀티 프로세스 : 하나의 프로세스가 다른 프로세스에 영향을 주지 않음., 멀티 스레드보다 많은 메모리 공간, CPU 시간 차지
